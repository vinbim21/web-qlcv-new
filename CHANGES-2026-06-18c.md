## 1. Thay đổi Database (Schema)

### Bảng `CatalogItem` — thêm cột mới

```sql
ALTER TABLE "CatalogItem" ADD COLUMN "projectGroupId" TEXT;
ALTER TABLE "CatalogItem" ADD CONSTRAINT "CatalogItem_projectGroupId_fkey"
  FOREIGN KEY ("projectGroupId") REFERENCES "ProjectGroup"("id") ON DELETE SET NULL;
CREATE INDEX "CatalogItem_projectGroupId_idx" ON "CatalogItem"("projectGroupId");
```

> **Mục đích:** Cho phép gắn hạng mục BIM Tools (Level 3 của nhóm PT) vào một Dự án (ProjectGroup).  
> `NULL` = không gắn dự án. Cascade: xóa ProjectGroup → `projectGroupId` về NULL (SET NULL).

### Bảng `Task` — thêm 2 cột mới

```sql
ALTER TABLE "Task" ADD COLUMN "pendingPlannedEnd"    TIMESTAMP;
ALTER TABLE "Task" ADD COLUMN "endChangeRequesterId" VARCHAR(30);
```

> **Mục đích:** Lưu yêu cầu đổi ngày kết thúc đang chờ duyệt.  
> - `pendingPlannedEnd`: ngày kết thúc mới mà người thực hiện đề xuất  
> - `endChangeRequesterId`: userId người gửi yêu cầu  
> Khi quản lý duyệt → `plannedEnd = pendingPlannedEnd`, xóa cả 2 cột.  
> Khi từ chối / hủy → chỉ xóa 2 cột, giữ nguyên `plannedEnd`.

## 2. Thay đổi Server Actions

### `src/server/actions/tasks.ts` — 5 action mới

| Action | Ai dùng | Mô tả |
|--------|---------|-------|
| `bulkSetPlannedStart` | Manager + Assignee | Đặt ngày bắt đầu hàng loạt. Manager cập nhật tất cả; Assignee chỉ cập nhật task mình được giao |
| `requestEndDateChange` | Manager + Assignee | Đề xuất đổi ngày kết thúc. Manager → đổi thẳng; Assignee → lưu vào `pendingPlannedEnd` |
| `approveEndDateChange` | Manager | Duyệt đề xuất: chép `pendingPlannedEnd` → `plannedEnd`, xóa pending |
| `rejectEndDateChange` | Manager + người gửi | Từ chối / hủy đề xuất: xóa `pendingPlannedEnd` |

### `src/server/actions/catalog.ts` — cập nhật 2 action

| Action | Thay đổi |
|--------|---------|
| `updateCatalogValue(id, value, parentId?, projectGroupId?)` | Thêm tham số `projectGroupId` (optional) |
| `batchSaveCatalogItems(workGroupId, level, parentId, values, projectGroupId?)` | Thêm tham số `projectGroupId` (optional) |

### `src/server/actions/timesheet.ts` — 1 action mới

| Action | Mô tả |
|--------|-------|
| `getTaskWeekEntries(taskId)` | Lấy danh sách giờ đã ghi của user trong tuần hiện tại (T2–CN) cho 1 task. Trả về `{ id, date, hours, note }[]`. Lưu ý: `hours` là `Decimal` của Prisma, phải convert bằng `Number(e.hours)` |

## 3. Thay đổi UI

### `/tasks` — Trang Công việc của tôi

#### 3a. Cột Bắt đầu
- **Bỏ** pencil inline + hidden `<input type="date">` (không hoạt động tốt)
- Hiển thị plain text, căn `align-top` (sửa lỗi lệch so với các cột khác)
- **Thêm nút "Đặt ngày bắt đầu"** vào thanh action bar dưới cùng (xuất hiện khi chọn ≥1 checkbox)
  - Click → modal chọn ngày → Áp dụng
  - Manager: cập nhật tất cả task đã chọn
  - Assignee: chỉ cập nhật task mình được giao (server tự lọc)

#### 3b. Cột Kết thúc — Workflow đề xuất đổi hạn

**Với Assignee (không phải quản lý):**
1. Chọn task → nút **"Đề xuất đổi hạn"** ở action bar
2. Nhập ngày mới → "Gửi yêu cầu"
3. Cột Kết thúc hiện badge cam **"→ DD/MM"** + nút ✕ (hủy yêu cầu)

**Với Manager:**
1. Thấy badge cam trong cột Kết thúc của task có pending
2. Click ✓ → duyệt (áp dụng ngay) / ✕ → từ chối
3. Hoặc dùng nút **"Đổi ngày kết thúc"** ở action bar → đổi trực tiếp không cần duyệt

#### 3c. Click dòng task — Modal chi tiết
- Click vào bất kỳ chỗ nào trên dòng (trừ checkbox/button/input) → mở modal
- Modal hiển thị:
  - Mã ID + tên công việc
  - **Nội dung công việc** (note)
  - **Bảng giờ tuần**: Ngày | Số giờ | Nội dung — fetch từ `getTaskWeekEntries`
  - Nút **"Ghi giờ"** → đóng modal, mở dialog ghi giờ
- Manager vẫn double-click để mở dialog sửa công việc

### `/admin/catalog` — Tab 3 "Dự án BIM Tools"

#### Nút "Quản lý dự án" (MỚI — trước "Quản lý loại hình")
- Mở modal quản lý danh sách dự án PT (`ProjectGroup` với `workGroupId = PT`)
- Mỗi dự án có: **Mã dự án** (uppercase, font mono) + **Tên dự án**
- Thao tác: Thêm (inline form) / Sửa (mở record dialog) / Xóa (có kiểm tra còn hạng mục không)

#### Form Thêm hạng mục — thêm trường Dự án
Thứ tự 3 trường:
1. **Dự án** (tùy chọn) — dropdown chọn từ danh sách PT projects
2. **Loại hình** (bắt buộc)
3. **Hạng mục** (bắt buộc, nhập nhiều dòng)

#### Form Sửa hạng mục — thêm trường Dự án
Tương tự form Thêm, có thể thay đổi Dự án / Loại hình / Hạng mục.

#### Cột "Dự án" trong bảng
- Trước: luôn hiển thị `—`
- Sau: hiện **mã dự án** (font mono), tooltip = tên đầy đủ
- Có **filter multi** (lọc theo dự án)

## 5. Hướng dẫn deploy / chạy lại

### Nếu DB chưa có cột mới (lần đầu pull commit này):

```bash
npx prisma db push
```

> Không cần `migrate` — dự án dùng `prisma db push` thay cho `prisma migrate dev`.

### Build & chạy local:

```powershell
# Dừng server cũ
Get-Process -Name "node" | Stop-Process -Force
Start-Sleep -Seconds 2

# Build
npm run build

# Start
npm start
```

1. Vào `/tasks` → click vào dòng task → kiểm tra modal chi tiết mở ra
2. Chọn ≥1 task → action bar xuất hiện các nút: Ghi nhận giờ / Đặt ngày bắt đầu / Đề xuất đổi hạn
3. Vào `/admin/catalog` → Tab 3 "Dự án BIM Tools" → kiểm tra nút "Quản lý dự án"
4. Thêm hạng mục → kiểm tra dropdown Dự án xuất hiện đầu tiên

## 6. Lưu ý kỹ thuật

- `hours` trong `TimeSheetEntry` là kiểu `Decimal` của Prisma → khi trả về JSON phải `Number(e.hours)`
- `pendingPlannedEnd` / `endChangeRequesterId` trên `Task`: chỉ dùng cho trang `/tasks`, không ảnh hưởng `/manage` hay `/assign`
- `projectGroupId` trên `CatalogItem`: chỉ dùng cho nhóm PT (BIM Tools), các nhóm khác để `NULL`
