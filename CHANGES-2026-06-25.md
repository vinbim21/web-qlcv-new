## 2. /manage — Tab "Dự án": Hiển thị group kể cả 0 việc

**Vấn đề:** Tree chỉ hiển thị các nhóm đã có task. Nếu Khai báo thông tin có Dự án / Loại hình / Hạng mục nhưng chưa giao việc → không thấy trong tree.

**Thay đổi:**
- Thêm `groupWorkGroupId: string | null` vào `ProjectOpt` (`assign-client.tsx`) và `task-lookups.ts` — biết project group thuộc workgroup nào.
- `catalogSeed` useMemo trong `manage-client.tsx`: build cây `dk → lk → Set<hk>` từ toàn bộ `projects` prop (bao gồm hạng mục chưa có task), lọc theo `activeWg`:
  - `groupWorkGroupId === null` → hiển thị ở tất cả tab (shared)
  - `groupWorkGroupId === activeWg` → chỉ hiện ở tab đó
- `byDuAn`, `byLoai`, `byHang` được pre-seed từ catalog trước khi fill task → g1/g2/g3 với 0 task vẫn xuất hiện với `(0 việc)`.
- `effectiveTreeCollapsed` cũng pre-collapse g3 từ catalog (hạng mục 0 việc bắt đầu đã thu gọn).
- Count g1 = `byLoai.size`, count g2 = `byHang.size` (tính trước khi push node) → đúng kể cả khi 0 task.

**Files:** `src/app/(app)/manage/manage-client.tsx`, `src/app/(app)/assign/assign-client.tsx`, `src/server/data/task-lookups.ts`

## 3. /admin/catalog — Xóa hạng mục kể cả có công việc

**Vấn đề:** Server action `deleteProject` throw error nếu hạng mục còn task. Client hiển thị `blockMsg` (chỉ có nút "Đóng") — admin không thể xóa.

**Thay đổi:**

### Server (`src/server/actions/projects.ts`)
- Bỏ `if (taskCount > 0) throw new Error(...)`.
- Thêm `prisma.task.updateMany({ where: { projectId: id, deletedAt: null }, data: { projectId: null } })` trước khi soft-delete project.
- Task **vẫn tồn tại**, chỉ mất liên kết `projectId` — admin có thể reassign sau.

### Client (`src/app/(app)/admin/catalog/catalog-client.tsx`)
- Đổi `blockMsg` → `warnMsg` ở 4 chỗ xóa hạng mục:
  - 2 chỗ xóa đơn (Bảng view + Dự án view)
  - 2 chỗ xóa hàng loạt (bulk bar)
- Dialog hiện cảnh báo *"X công việc sẽ mất liên kết dự án"* nhưng vẫn có nút **Xóa**.
- `blockMsg` (chặn hoàn toàn) chỉ còn dùng cho xóa Dự án (ProjectGroup) có hạng mục con.
