# CHANGES-2026-06-23c

## Catalog / Khai bao thong tin

- Reworked tab `2. Du an` grouped view from a nested card/tree into a table-aligned tree.
- Group hierarchy is now: `Du an -> Loai hinh -> Hang muc -> Khoi/He thong`.
- Added table headers for `Du an`, `Loai hinh`, `Hang muc`, `Khoi/He thong`, `Bat dau`, `Dong goi`, `Quy mo`, `Thao tac`.
- Moved `Bat dau` and `Dong goi` to the `Hang muc` group row because these dates are synced per hang muc across block systems.
- Hidden empty block-system child rows when a hang muc has no `blockSystem`; edit/delete actions stay on the hang muc row.
- Reduced widths for `Du an`, `Loai hinh`, and `Hang muc`; expanded `Khoi/He thong`.
- Project and construction type group rows show only codes, with names available as tooltips.
- Added header checkbox for select all in grouped project view.
- Bulk action bar in grouped project view now uses a soft blue fixed bottom style, aligned with the manage page selection bar.
- Selection now expands related tree branches; collapse/expand buttons affect selected branches when rows are selected, otherwise the whole tree.

## Reports

- Report task list now falls back to declared catalog data for tasks without direct `projectId`, including BIM Tools tasks.
- BIM Tools report rows can now resolve `Du an`, `Loai hinh`, and `Hang muc` from `CatalogItem.projectGroup` and parent level 2 catalog.
- Added `Giai doan` column after `Cong viec` in the report task list.
- `Giai doan` participates in searching, filtering, sorting, and display.

## Tasks / Manage

- Task and manage views now include requester names for delete and deadline-change requests where needed by badges/tooltips.
- Delete-request badges were restyled to a lighter inline presentation.
- Pending deadline request tooltip includes requester name and note.

## Verification

- Ran production build successfully with `npm run build`.
- Restarted production server with `npm start`.
- Verified `/admin/catalog` and `/reports` return HTTP 200.
