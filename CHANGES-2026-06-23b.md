## 4. Luồng xóa công việc (xóa thẳng vs đề xuất xóa)

### Schema (`prisma/schema.prisma`)
- `NotificationType`: thêm `TASK_DELETE_REQUESTED`
- `Task`: thêm `deleteRequestedAt DateTime?`, `deleteRequesterId String?`, `deleteRequestNote String?`, `endChangeNote String?`
- `User`: thêm relation `deleteRequests Task[] @relation("TaskDeleteRequester")`

### Server Actions (`tasks.ts`)
- `deleteTask(id)`: Assignee xóa được khi task **chưa duyệt** (`startApprovedAt null`); Manager xóa mọi lúc
- `requestDeleteTask(id, note)`: Assignee đề xuất xóa task **đã duyệt** → set `deleteRequestedAt` + notify người duyệt (hoặc `notifyManagers` nếu không có approverId)
- `approveDeleteTask(id)`: Manager duyệt → soft delete + notify người đề xuất
- `rejectDeleteTask(id)`: Manager từ chối / User hủy → clear request + notify (chỉ khi manager từ chối)
- `requestEndDateChange()`: thêm param `note` (lý do dời hạn); clear `endChangeNote` khi duyệt/từ chối

### UI — `/tasks` (`tasks-client.tsx`)
- `TaskRow` type: thêm 4 fields (`endChangeNote`, `deleteRequestedAt`, `deleteRequesterId`, `deleteRequestNote`)
- **Bar hành động**: nút **Xóa** hiện khi chọn 1 task (phải là assignee hoặc manager, chưa pending delete)
- **Dialog xóa** (`DeleteTaskDialog`):
  - Task **chưa duyệt**: xóa thẳng (nút đỏ "Xóa")
  - Task **đã duyệt** + non-manager: hiện tên người duyệt *"gửi tới [Tên]"* (fallback "quản lý" nếu không có), textarea lý do tùy chọn, nút amber "Gửi đề xuất xóa"
- **Cột Clock**: khi task có `deleteRequestedAt` → thay icon đồng hồ bằng badge "Chờ duyệt xóa" + nút hành động:
  - Assignee: nút × hủy
  - Manager: nút ✓ duyệt + ✗ từ chối
- **Badge đề xuất dời hạn**: tooltip hiện lý do (`endChangeNote`)

### UI — `/manage` (`manage-client.tsx`)
- `ManageTaskRow` type: thêm 4 fields, `deleteRequesterName`
- **Badge trong dòng**: khi `deleteRequestedAt` → hiện "[Tên] xin xóa — [lý do]" (màu đỏ nhạt) + nút ✓/✗ cho manager
- **Bar hành động bulk**: nút **Duyệt xóa** xuất hiện có điều kiện khi ≥1 task chọn có `deleteRequestedAt` (xử lý tuần tự, toast kết quả)
- **Filter "Chưa giao/Chưa duyệt"**:
  - KPI count `unassignedOrPending`: cộng thêm task có `deleteRequestedAt`
  - Quick filter `CHUA_GIAO`: hiện task có `deleteRequestedAt` (bypass điều kiện ẩn)
  - `kpiBase` period filter: bypass cho task có `deleteRequestedAt` (không bị ẩn bởi lọc tuần/tháng)
- **Ngày định dạng dd/mm/yyyy** (`fmtDate`): sửa 4 chỗ còn hiện ISO raw — `deadlineLabel()`, tooltip "Trễ hạn", badge "Xin dời →"

### Data pipeline (`manage/page.tsx`, `tasks/page.tsx`)
- Include `deleteRequester { select: { fullName: true } }` trong query
- Map `deleteRequestedAt`, `deleteRequesterId`, `deleteRequesterName`, `deleteRequestNote` vào TaskRow
